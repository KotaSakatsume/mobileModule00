## なぜ(背景・設計判断)

- **依存追加ゼロ / Material のみ**: ex00 の要件は1ページ1ボタンに収まる最小構成。状態管理・ルーティング・テーマ設計は過剰なため導入しない。`ElevatedButton` が標準で揃う Material を採用し、Piscine 採点機の環境差異に弱くしないため `pubspec.yaml` のバージョンを過度にピン留めしない。
- **StatefulWidget 採用**: ex00 単体なら Stateless でも足りるが、後続 ex で状態が乗ることを見越し、ボタンイベントハンドラを持つ `State` を持たせる方が自然と判断。
- **`debugPrint` を採用(`print` 不採用)**: `print` は `avoid_print` lint で警告になり得るため。出力先は console のみで UI には一切出さず、文言は完全一致で `Button pressed`。
- **固定座標を使わない中央寄せ**: 水平は `Center` + `Column` デフォルト、垂直は `Column(mainAxisAlignment: center)`。固定 px を避けることで回転・異サイズでもレイアウトが破綻しない。
- **テストの `debugPrint` 復元は try/finally**: レビュー提案の `addTearDown(() => debugPrint = original)` は、`flutter_test` が「foundation の debug 変数はテアダウンより前に復元済み」を不変条件として検証するため失敗する。テスト本体内で同期的に復元する形へ調整した(検証観点・スコープは不変)。
